<objective>
## Goal
Detect frontmost macOS app at key press time and reshape transcription via LLM using a per-app prompt before pasting. Opt-in via config. Command mode skips auto-adapt entirely.

## Purpose
Users dictating into Slack want casual tone; into Mail want formal prose. Auto-adapt reshapes the transcription automatically without user intervention beyond one-time config setup.

## Output
- `src/local_whisper/auto_adapt.py` — module with app detection, config parsing, LLM call, built-in presets
- `src/local_whisper/app.py` — capture active app at press time; wire `auto_adapt.apply()` in dictation path
- `tests/test_auto_adapt.py` — unit tests covering opt-in gate, preset matching, LLM path, passthrough, failure modes
</objective>

## AC-1: Opt-in gate
```gherkin
Given config.toml has no [auto_adapt] section (or enabled = false)
When auto_adapt.apply() is called
Then text is returned unchanged and no LLM call is made
```

## AC-2: Built-in Slack preset
```gherkin
Given [auto_adapt] enabled = true in config and app_name = "Slack"
When auto_adapt.apply() is called with dictated text
Then LLM is called with the built-in casual/emoji Slack prompt and the result is returned
```

## AC-3: Built-in Mail preset
```gherkin
Given [auto_adapt] enabled = true in config and app_name = "Mail"
When auto_adapt.apply() is called with dictated text
Then LLM is called with the built-in formal email prompt and the result is returned
```

## AC-4: Config override
```gherkin
Given [auto_adapt] enabled = true and a [auto_adapt.miro] sub-section with app = "Miro" and custom prompt
When auto_adapt.apply() is called with app_name = "Miro"
Then LLM is called with the custom prompt from config
```

## AC-5: Unknown app passthrough
```gherkin
Given [auto_adapt] enabled = true and app_name not matched by any preset or config section
When auto_adapt.apply() is called
Then text is returned unchanged and no LLM call is made
```

## AC-6: Command mode excluded
```gherkin
Given any config state
When Right Command is held with text selected (command mode)
Then auto_adapt.apply() is never called
```

## AC-7: Active app captured at press time
```gherkin
Given user holds Right Command in dictation mode
When _on_key_press fires
Then get_active_app() is called immediately and the result stored for use in _record_and_process
```

## AC-8: LLM failure → passthrough
```gherkin
Given [auto_adapt] enabled = true and a matching preset
When the OpenAI API call raises an exception
Then original text is returned unchanged (no crash, error printed to stderr)
```

<task type="auto">
  <name>Task 1: Create auto_adapt.py module</name>
  <files>src/local_whisper/auto_adapt.py</files>
  <action>
    Create module following the auto_cleanup.py pattern (`_is_enabled` + `apply`).

    **Functions to implement:**

    `get_active_app(path: Path = _CONFIG_PATH) -> str`
    - Use `NSWorkspace.sharedWorkspace().frontmostApplication().localizedName()` via PyObjC
    - Import pattern: `try: from AppKit import NSWorkspace as _NSWorkspace; _HAS_APPKIT = True` (match command.py import guard)
    - Return "" on ImportError or any exception (fail-open)

    `_get_prompt(app_name: str, config_data: dict) -> str | None`
    - Iterate all sub-sections of config_data (keys not equal to "enabled")
    - Each sub-section is a dict with keys `app` (str) and `prompt` (str)
    - Return prompt if sub-section's `app` matches app_name (case-insensitive)
    - If no config match, check `_BUILTIN_PROMPTS` dict
    - Return None if no match

    `apply(text: str, app_name: str = "", path: Path = _CONFIG_PATH) -> str`
    - Config check: read `[auto_adapt]` section; if `enabled` not True → return text unchanged
    - Config read per call (no caching) — consistent with auto_cleanup pattern
    - If app_name empty → return text unchanged
    - Call `_get_prompt(app_name, config_section)` → if None → return text unchanged
    - Call OpenAI API using same pattern as command.py:
      - `api_key = os.environ.get("LOCAL_WHISPER_OPENAI_API_KEY")`
      - If no key → print warning to stderr, return text unchanged
      - Model: `os.environ.get("LOCAL_WHISPER_COMMAND_MODEL", "gpt-4o-mini")`
      - Base URL: `os.environ.get("LOCAL_WHISPER_OPENAI_BASE_URL")`
      - System prompt: "Reformat the following transcription. Return only the reformatted text — no explanation, no preamble."
      - User content: f"{prompt}\n\n{text}"
      - `max_completion_tokens=1024`
    - On any exception → print to stderr, return original text (AC-8)

    **Built-in presets constant** (module-level dict):
    ```python
    _BUILTIN_PROMPTS: dict[str, str] = {
        "Slack": "Rewrite as casual Slack message. Use emojis where natural. Short sentences.",
        "Mail": "Rewrite as formal professional email. Correct grammar, clear paragraphs, no casual language.",
    }
    ```

    **Config structure expected:**
    ```toml
    [auto_adapt]
    enabled = true

    [auto_adapt.slack]
    app = "Slack"
    prompt = "Custom slack prompt"

    [auto_adapt.miro]
    app = "Miro"
    prompt = "Bullet points for diagram notes"
    ```

    Avoid: caching config between calls, crashing on malformed config, importing openai at module level (lazy import inside apply)
  </action>
