  <files>src/local_whisper/hotkey.py</files>
  <action>
    Create `src/local_whisper/hotkey.py` with a `HotkeyListener` class:

    ```python
    class HotkeyListener:
        def __init__(self, on_activate: Callable[[], None], on_deactivate: Callable[[], None]) -> None:
            """Listen for Right Command hold/release globally.

            Args:
                on_activate: Called when Right Command is pressed.
                on_deactivate: Called when Right Command is released.
            """

        def start(self) -> None:
            """Start the keyboard listener in a background thread."""

        def stop(self) -> None:
            """Stop the keyboard listener."""
    ```

    Implementation:
    - Use `pynput.keyboard.Listener(on_press=..., on_release=...)`
    - Listen for `keyboard.Key.cmd_r` specifically
    - `_pressed: bool` flag — set True on press, False on release
    - Only call `on_activate` if not already pressed (debounce AC-5)
    - Only call `on_deactivate` if was pressed
    - Start listener with `daemon=True` so it doesn't block process exit
    - Print macOS Accessibility permission hint to stderr if listener fails to start

    Avoid: blocking the main thread in `start()` — listener runs in its own thread.
    Avoid: catching all exceptions broadly — let permission errors surface clearly.
  </action>
  <verify>uv run python -c "from local_whisper.hotkey import HotkeyListener; print('import OK')"</verify>
  <done>AC-1 and AC-5 satisfied: listener imports, debounce logic present</done>

    ```python
    def write_and_paste(text: str) -> None:
        """Write text to clipboard and paste it at the active cursor position.

        Uses pyperclip to write to clipboard, then osascript to send Cmd+V.

        Args:
            text: The text to paste.
        """
    ```

    Implementation:
    - `pyperclip.copy(text)` to write to clipboard
    - `subprocess.run(["osascript", "-e", 'tell application "System Events" to keystroke "v" using command down'], check=True)`
    - Small `time.sleep(0.05)` before osascript — give clipboard time to settle
    - Log "Pasted N chars" to stderr

    Also add a standalone `copy(text: str) -> None` that only writes to clipboard (no paste):
    - Used as fallback if osascript fails
    - Useful for testing without active text field

    Avoid: using pbcopy (macOS-only shell, subprocess overhead) — pyperclip is cross-process and already installed.
    Avoid: assuming paste always succeeds — if osascript fails, fall back to copy-only and warn to stderr.
  </action>

<task type="auto">
  <name>Task 3: App loop + update CLI entry point</name>
  <files>src/local_whisper/app.py, src/local_whisper/__main__.py</files>
  <action>
    **app.py** — main application orchestrator:

    ```python
    class App:
        """Orchestrates hotkey → record → transcribe → paste flow."""

        def __init__(self) -> None:
            self._recording = False
            self._audio_data: np.ndarray | None = None
            self._listener = HotkeyListener(
                on_activate=self._on_key_press,
                on_deactivate=self._on_key_release,
            )

        def run(self) -> None:
            """Start the app. Blocks until Ctrl+C."""

        def _on_key_press(self) -> None:
            """Start recording in a background thread."""

        def _on_key_release(self) -> None:
            """Stop recording, transcribe, paste — in a background thread."""
    ```

    Key implementation details:
    - `_on_key_press`: set `_recording = True`, start `threading.Thread(target=self._record_loop, daemon=True)`
    - `_record_loop`: calls `audio.record_until_stop()` — see note below
    - `_on_key_release`: set `_recording = False` (signals _record_loop to stop), then transcribe + paste in thread
    - Use `threading.Event` to signal recording stop — cleaner than polling bool
    - On startup: print "local-whisper running. Hold Right ⌘ to dictate. Ctrl+C to quit." to stderr

    **Audio change needed:** `audio.py` must support event-based stopping, not fixed duration.
    Add to `audio.py`:
    ```python
    def record_until_event(stop_event: threading.Event, sample_rate: int = 16000) -> np.ndarray:
        """Record audio until stop_event is set."""
    ```
    Use `sounddevice.InputStream` with a callback that appends chunks to a list.
    On `stop_event.wait()`, close the stream and concatenate chunks.

    **__main__.py** — add `--run` flag:
    - `--run`: instantiate `App()` and call `.run()` (blocks, Ctrl+C to exit)
    - `--test`: existing behaviour (fixed 5s record → transcribe → print)
    - No args: print help

    Avoid: calling transcription on the main thread — keep UI responsive.
    Avoid: global state — all state lives in `App` instance.
  </action>
  <verify>uv run python -m local_whisper --help | grep -E "(run|test)" && echo "CLI flags OK"</verify>
  <done>AC-2, AC-4 satisfied: App class wires all modules, --run flag exists</done>
