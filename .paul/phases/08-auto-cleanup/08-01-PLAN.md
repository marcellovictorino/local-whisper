<objective>
## Goal
Post-process every dictation transcription to remove common filler words and collapse
immediate word repetitions before paste. Always-on by default; opt-out via config.

## Purpose
Raw ASR output is verbatim — it includes "um", "uh", "you know", and stuttered
repetitions. Users who dictate prose, messages, or notes don't want those in the
pasted result. Rule-based cleanup is offline, zero-latency, and zero-dependency.

## Output
- `src/local_whisper/auto_cleanup.py` — `apply()` + `_is_enabled()` functions
- `src/local_whisper/app.py` — `auto_cleanup.apply()` inserted in `_record_and_process()`
- `tests/test_auto_cleanup.py` — unit tests for both functions
</objective>

<acceptance_criteria>

## AC-1: Filler words removed from dictation output
```gherkin
Given a transcription containing filler words (um, uh, er, ah, hmm, you know)
When _record_and_process() runs
Then pasted text contains none of those filler words
And surrounding words are joined with a single space (no double spaces)
```

## AC-2: Immediate word repetitions collapsed
```gherkin
Given a transcription with an immediately repeated word ("I I need to go")
When _record_and_process() runs
Then pasted text has the word once ("I need to go")
And non-adjacent repetitions ("the cat and the dog") are NOT collapsed
```

## AC-3: Config opt-out disables cleanup
```gherkin
Given ~/.config/local-whisper/config.toml contains [auto_cleanup] with enabled = false
When _record_and_process() runs
Then pasted text is the raw transcription with no filler or repetition removal
```

## AC-4: Enabled by default with no config
```gherkin
Given ~/.config/local-whisper/config.toml does not exist (or has no [auto_cleanup] section)
When _record_and_process() runs
Then cleanup runs as if enabled = true
```

## AC-5: Command mode not affected
```gherkin
Given the user is in command mode (_command_record_and_process)
When voice_cmd is transcribed
Then auto_cleanup is NOT applied to voice_cmd
And the command instruction reaches the LLM verbatim
```

<task type="auto">
  <name>Task 1: Create auto_cleanup module</name>
  <files>src/local_whisper/auto_cleanup.py</files>
  <action>
    Create `src/local_whisper/auto_cleanup.py` with two functions:

    `_is_enabled(path: Path = _CONFIG_PATH) -> bool`
    - Open config.toml if it exists; parse `[auto_cleanup]` section
    - Return `data.get("auto_cleanup", {}).get("enabled", True)`
    - Return True on missing file, missing section, or parse error (fail open)
    - No stderr noise for missing file — it's the expected default state

    `apply(text: str, path: Path = _CONFIG_PATH) -> str`
    - Call `_is_enabled(path)` first; return `text` unchanged if False
    - Step 1 — collapse immediate word repetitions (case-insensitive):
      regex `r'(\b\w+\b)(\s+\1)+\b'` with `re.IGNORECASE`, replace with `r'\1'`
      Only collapses adjacent duplicates; non-adjacent repeats are untouched.
    - Step 2 — strip filler words from this exact set (no more, no less):
      `{"um", "uh", "er", "ah", "hmm", "you know"}`
      Match whole words/phrases, case-insensitive:
      - Single-word fillers: `r'\bum\b'`, etc.
      - Multi-word: `r'\byou\s+know\b'`
      After removal, collapse multiple spaces to one and strip leading/trailing whitespace.
    - Return cleaned text
    - On any exception: log to stderr, return original text (never crash the pipeline)

    Avoid: stripping "like", "so", "right", "well" — too context-dependent and would
    corrupt legitimate speech. Keep the filler set conservative.
  </action>
  <verify>
    python -c "from local_whisper.auto_cleanup import apply; print(repr(apply('Um so I I need to go you know')))"
    Expected: 'so I need to go'
    (or similar — "so" preserved, "Um" stripped, "I I" collapsed, "you know" stripped)
  </verify>
  <done>AC-1, AC-2, AC-3, AC-4 satisfied by the module logic</done>
</task>
